Hexo部署至云服务器

前言

由于将博客部署到Github Pages这种托管服务着实不太方便，目前我所尝试的三家都有一些让人不太爽的方面：

  • Github Pages：一是速度比较慢，二是Github禁止百度的爬虫，用百度是搜不到自己博客的。
  • CODING Pages：一向不太稳定，老是冗机。
  • Gitee自定义域名又需要收费，一年99都快赶上学生服务器了。

于是博主一狠心，借着自己学生的身份，购买了轻量应用服务器（学生优惠太香了，一折还可行），决定将自己的博客部署再到云服务器上。这样不仅可以提高博客的稳定性，同时还可以在操作的过程中慢慢了解学习服务器和Linux的操作。本人也是纯纯的小白，如有写的不对的地方希望大家能帮我指出来，现在这里谢谢大家啦！诶嘿！

一、环境

本地环境

  • macOS(11.5.2)
  • 已装好 Git、Node.js 和 Hexo
  • 已在 Github Pages 部署了 Hexo 播客

服务器环境

  • 腾讯云SAS（ 1核2G, 6M, Ubuntu 20.4LTS ）

二、本地机器配置

如果已经将Hexo部署到 GitHub Pages / Gitee Pages / CODING Pages 上了，那么你的本地机器上已经完成了配置，这里不需要特别配置。

若是未曾部署过Hexo，请先参考我的另一篇文章基于Hexo+Github个人博客搭建完成Hexo的部署。文章内是将Hexo部署到Github进行托管，而我们这里是将Hexo部署到我们的服务器上。你可以先完全按照那篇文章进行部署，随后根据本篇博客将Hexo部署到服务器并作略微修改即可。（需要修改的地方会写在文末，只是一些细微的修改，请放心食用）

  • macOS推荐用Termius连接服务器。
