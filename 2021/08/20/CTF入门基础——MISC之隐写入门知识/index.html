CTF入门基础——MISC之隐写入门知识

MISC-隐写

01.图片隐写

1.基本套路

  1. 将信息隐藏于图片属性
  2. 数据类型的隐藏
  3. 文件嵌套文件
  4. 文件内部的冗余信息
  5. 基于图片本身的数据隐藏

2.具体内容

  1. 将信息隐藏于图片属性：

    • 查看图片属性得到相关信息
    • 有可能藏在EXIF信息里面，如GPS，拍摄相机等信息

  2. 数据类型的隐藏：

    • 通常会打不开文件！

    • 出现原因：

      • 缺少文件头
      • 文件头与后缀不匹配

    • 工具：

      • WinHex
      • 010Editor

    • 常见文件头

      | 文件类型 | 文件后缀 | 文件头 |
      | :———————: | :———: | :——————: |
      | JPEG | .jpg | FFD8FFE1 |
      | PNG | .png | 89504E47 |
      | GIF | .gif | 47494638 |
      | TIFF | .tif | 49492A00 |
      | Windows Bitmap | .bmp | 424DC001 |
      | ZIP Achieve | .zip | 50B0304 |
      | RAR Achieve | .rar | 52617221 |
      | Adobe Photoshop | .psd | 38425053 |
      | Rich Text Format | .rtf | 7B5C727466 |
      | XML | .xml | 3C3F786D6C |

      | Adobe Arcobat | .pdf | 255044462D312E |
      | Wave | .wav | 57415645 |
      | pcap | .pcap | 4D3C2B1A |

  3. 文件嵌套文件

    • 出现原因：

      • 文件头的嵌套
      • copy等命令的使用

    • 工具：

      • WinHex（010Editor）
      • Binwalk（Foremost）

    • 有时会存在一个文件的全部信息夹在另一个文件中

      | JPEG 1 | 信息 |
      | :——: | :———: |
      | JPEG 1 | 头部信息 |
      | JPEG 2 | 全部信息 |
      | JPEG 1 | 尾部信息 |

  4. 文件内部的冗余信息

    • 出现原因：
      • 后续加入的不影响图片正常显示的信息
    • 工具：
      • WinHex（010Editor）
      • strings

  5. 基于图片本身的数据隐藏

    • 出现原因：

      • 通过后期对于图片宽、高进行调整，实现隐藏部分图片内容，从而达到数据隐写的目的。一般来说，这种题目会有很明确的提示，譬如图片的宽和高明显不对等或指向某方向的箭头等

    • LSB隐写：

      • 原理：

        • 图片的图像像数一般都是由RGB三原色（红绿蓝）组成， 每一种颜色占8位，取值为0x00—0xFF，就是256种，一共包含 了256的3次方种颜色，即16777216种颜色 但是人类的眼睛只 可以分辨大概1000万种不同的颜色，还有很多种细微的变化人 类的眼睛是觉察不到的，通过改变人眼观察不到的颜色数据来 进行数据隐写

      • LSB隐写只会出现在png或bmp等无损的图片格式中

      • GIF文件处理：

        • 利用StegSolve对gif文件进行逐帧分析

        • 一些类型：

          1. 逐帧分析后将图片合并得到flag

          2. 逐帧分析后将图片合并得到二维码

      • StegSolve的使用：

        1. 准备：

          • StegSlove软件
          • Java开发环境

        2. 使用说明

          • 程序主界面：

          • File

            • Open————————————>打开文件
            • Save As———————————>保存文件
            • Exit—————————————>退出

          • Analyse

            • File Format——————————>文件格式信息

            • Data Extract —————————>数据提取

              • Bit Planes——>ARGB颜色通道，数字越高，该通道颜色亮度越大
              • Extra By———>分为行和列
              • Bit Order———>位顺序序：MSB是一串数据的最高位，LSB是一串数据的最低位
              • Bit Plane Order——>位平面顺序:整个图像分解为8个位平面，从LSB到MSB。随着从位平面0 到位平面7，位平面图像的特征逐渐变得复杂，细节不断增加。

            • Stereogram Solver———————>立体试图，控制左右偏移

            • Frame Browser ————————>逐帧浏览器，用于查看GIF等

            • Image Combiner————————>图片拼接

02.音频隐写

  1. 摩斯密码

    摩尔斯电码是一种时通时断的信号代码，通过不同的排列顺序来表达不同的英文字母、数字和标点符号。部分音频隐写通过快速播放的Morse电码来传递信息（也可能是正常速度的）

    • 解题方法：
      • 通过Audacity打开音频文件，放大波形图，来判断Morse电码
      • 通过CTFcrack或其他工具进行解码

  2. 频谱图隐藏信息

    频谱是频率谱密度的简称，是频率的分布曲线。复杂振荡分解为振幅不同和频率不同的谐振荡。这些谐振荡的幅值按频率排列的图形叫做频谱。在CTF中，我们可以单独只对一个声 道中，隐写进信息

    • 解题方法：

      通过Audacity打开音频文件，选择音频所在音轨，”频谱”——>”频谱图”即可

  3. MP3文件的隐写

    MP3stego是著名的音频数据隐写工具，支持常见的压缩音频文件格式如mp3的数据嵌入，它采用的是一种特殊的量化方法并且将数据隐藏在MP3文件的奇偶校验块中

    • 解题方法：

      使用MP3stego的decode模块即可
