Java入门笔记1——类与对象

Java入门笔记1——类与对象

1.0 基本概念

  1. 面向对象程序设计：

    面向对象编程——Object Oriented Programming，简称OOP，是一种程序设计思想。

    所谓面向对象，就是将客观事物看作具有状态和行为的对象，通过抽象找出同一类对象的共同状态和行为，构成类。可以提高程序的复用性和可扩展性。

    在面向对象编程的过程中，对象是类的实例，是程序的基本单元，而程序和数据则被封装在对象中，从而提高软件的灵活性和扩展性。维基百科中这样写道：

    通常，OOP被理解为一种将程序分解为封装数据及相关操作的模块而进行的编程方式。

    由此，我们可以看到，面向对象编程的核心就是对类和对象的运用，这两个概念我们会在下面详细举例说明，这里不理解也没关系。

例如，我们要设计一个学生管理系统，那么我们就可以抽象出学生这个类，而通过这个类创建的对象则是每个具体的学生。学生这个类有一定的共性，例如他们都应该有姓名、性别、年龄、年级和班级等属性，此外还有听讲、回答问题、写作业、运动、玩耍、考试等行为。

  1. JVM简介

    • 类加载器——Class Loader：加载类文件到内存

    • 执行引擎——Execution Engine：也叫解释器，负责解释命令，交由操作系统执行

    • 本地库接口——Native Interface：本地接口的作用是融合不同的语言为java所用

    • 运行时数据区——Runtime Data Area：即JVM的内存

      • 方法区——Method Area：用于存储已被虚拟机加载的类信息、常量、静态变量以及即时编译器编译后的代码等数据。

      • 堆——Heap：堆是Java对象的存储区域。任何用new字段分配的Java对象实例和数组，都被分配在堆上。

      • 虚拟机栈——VM Stack：虚拟机栈中执行每个方法的时候，都会创建一个栈桢用于存储局部变量表——各类基本变量，操作数栈，动态链接，方法出口等信息。
      • 本地方法栈：执行每个本地方法的时候，都会创建一个栈帧用于存储局部变量表、操作数栈、动态链接以及方法出口等信息。
      • 程序计数器。指示Java虚拟机下一条需要执行的字节码指令。

  2. JDK/JRE/JVM/JSR/JCP：

    • JVM：Java Virtual Machine，Java 虚拟机。

    • JRE：Java Runtime Environment，包含 JVM 及 Java 核心类库。简单来说JRE就是运行Java字节码的虚拟机。

    • JDK：Java Development Kit。如果只有Java源码，要编译成Java字节码，就需要JDK，因为JDK除了包含JRE，还提供了编译器、调试器等开发工具。由此可见，JDK是包含JRE的。
    • JSR：Java Specification Request，为了保证Java语言的规范性，SUN公司搞了一个JSR规范。
    • JCP：Java Community Process，负责审核JSR的组织就是JCP组织。

  3. Java中数据的存储

    • Register：最快的存储设备，但是程序员无法调用
    • Stack：存在于RAM（Random Access Memory）中，Java的基本数据类型存储在此
    • Heap：灵活且方便，可以存储不定长度的数据，主要用来存储Java中的对象，但速度略慢于Stack
    • Constant storage：常量存储在程序中。
    • Non-RAM stroage：将对象存储为可以存在于其他介质上的形式，需要的时候再转换为RAM可以运行的形式

  4. 基本类型及其包装类型

    基本类型都有一个包装类型，包装类型可以作为类来申请相应类型的对象，例如整型int的包装类型为Integer。

    • 基本类型自动转化为包装类型（自动装箱）

    • 包装类型转化为基本类型（自动拆箱）

1.1 类与对象

  1. 类与对象

    • 类：定义了一件事务的抽象特点。类的定义包含了数据的形式以及数据的形式。

  3. 封装

    所谓封装，就是把数据和对数据的操作放在一起。其实也就是将属性和方法封装在一起（封装在类中）。

    • 封装有两层含义：

      • 一是把对象的属性和服务看作一个密不可分的整体，将这两者封装在一个不可分割的独立单元中。
      • 二是指信息隐藏，将信息和数据分级管理：外界不需要知道的信息隐藏起来，外界需要知道的信息又按照是否能由外界修改分为两类。这样则可以尽可能将对象功能的一些细节隐藏起来，提高安全性。

    • 封装的优点：

      • 良好的封装能够减少耦合，符合程序设计追求“高内聚，低耦合”
      • 类内部的结构可以自由修改
      • 可以对成员变量进行更精确的控制
      • 隐藏信息实现细节

  4. OOP特性：

    • 一切都是对象

    • 程序就是一堆相互发消息的对象

    • 每个对象有自己的存储空间，里面是其他的对象
    • 每个对象都有一个类型
    • 所有属于某个特定类型的对象可以提供相同的服务

1.2 定义类

一个类由两种东西组成：表示对象有什么的属性/特征的成员变量和表示对象能做什么的成员函数。

一旦定义了类，我们就可以创建这个类的多个对象，这些对象都会做那个类所定义的动作（函数），但是各自具有不同的数据。

首先，我们定义一个类，需要知道两件事：

  1. 这个类它有哪些属性？
  2. 这个类它有哪些功能或者说操作？

1.3 成员变量和成员函数

  1. 成员变量
    • 类定义了对象中所具有的所有变量，这些变量称为成员变量，例如实例(2)中的price，balance 和 total
    • 每个对象都有自己的变量，和同一个类的其他对象的变量是分开的，即每个对象都以一套符合该类的变量
  2. 成员函数
    • 类定义了对象中所需要的所有操作，即函数，这些函数称为成员函数
  3. this变量
    • this是成员函数的一个特殊的固有的本地变量，它表达了调用这个函数的那个对象本身，例如若是 vm 调用类中的方法，则 this 指的就是 vm 这个对象本身。
  4. 调用成员函数
    • 通过 . 运算符则可以调用某个对象的成员函数
    • 在成员函数内部可以通过 this 直接调用自己或者其他成员变量函数

1.4 函数重载

  • 创建对象的时候给出不同的参数值，则会自动调用不同的构造函数来创建对象
